## 3. Architecture Validation Report

### System Boundaries

**README-ARCH-001 — Provider hierarchy ordering not documented (carry-over)**
**Severity:** Minor
**README Location:** Lines 47–79 — theming and localization shown in separate Quick Start sections, no nesting order stated.
**Evidence:** Architecture integration contract (`docs/raw/architecture/integration-contracts/getting-started.md:7-14`): Provider nesting is fixed — `ThemeProvider` outermost, `LanguageProvider` inside, consumer providers inside both.
**Recommendation:** Add a provider hierarchy note to Quick Start: "Provider ordering is fixed: `ThemeProvider` must wrap `LanguageProvider`."

**Evidence:** `docs/raw/feature/templates/README.md:12`: `src/types/            ← template.types.ts interfaces`
**Recommendation:** Add `types/` to the project structure listing.

---

### Architectural Drift

**README-ARCH-003 — Spacing base unit inconsistency (partial resolution)**
**Severity:** Minor
**README Location:** Line 123: "Spacing: 4px base unit"
**Evidence:** `docs/raw/feature/theming/tokens.md:7`: "Base unit: 8px (MUI standard). Note: The internal token value `0.5` = 4px = `0.5×` when the base unit is 8px." The README's "4px base unit" was interpreted as wrong in the prior audit, but is now clarified — the base unit is 8px, the smallest token (`internal`) equals 4px. README should reflect 8px base to be accurate.
**Recommendation:** Change to "Spacing: 8px base unit — `spacing.internal` = 4px, `spacing.xs` = 8px, `spacing.md` = 16px"
